# Google Sheetsセットアップガイド

タスク管理アプリをGoogle Sheetsと連携して、複数デバイス間でデータを共有する方法を説明します。

## 概要

このアプリは、Google Sheetsをバックエンドデータベースとして使用します。Google Apps Scriptをウェブアプリとしてデプロイすることで、どのデバイスからアクセスしても同じタスクデータが表示されます。

## 必要なもの

- Googleアカウント
- Google Sheets
- Google Apps Script

## セットアップ手順

### 1. スプレッドシートの作成

1. [Google Sheets](https://sheets.google.com/)にアクセス
2. 新しいスプレッドシートを作成
3. スプレッドシート名を「タスク管理」などに変更（任意）

### 2. Google Apps Scriptのデプロイ

1. スプレッドシートで「拡張機能」→「Apps Script」を選択
2. 新しいプロジェクトが開きます
3. `google-apps-script.js` ファイルの内容を全てコピー
4. Apps Scriptエディタに貼り付け（既存のコードを置き換え）
5. プロジェクト名を「TaskApp Backend」などに変更（任意）
6. 「保存」ボタンをクリック

### 3. Web アプリとしてデプロイ

1. 「デプロイ」→「新しいデプロイ」をクリック
2. 「種類の選択」で⚙️アイコンをクリック→「ウェブアプリ」を選択
3. 設定を以下のように入力：
   - **説明**: 「TaskApp API v1」（任意）
   - **次のユーザーとして実行**: 「自分」
   - **アクセスできるユーザー**: 「全員」
4. 「デプロイ」ボタンをクリック
5. 初回デプロイ時は承認が必要です：
   - 「アクセスを承認」をクリック
   - Googleアカウントを選択
   - 「詳細」をクリック
   - 「（プロジェクト名）（安全ではないページ）に移動」をクリック
   - 「許可」をクリック
6. **ウェブアプリURL**が表示されます（重要！）
7. URLをコピーして保存しておきます
   - 形式: `https://script.google.com/macros/s/xxxxx/exec`

### 4. タスクアプリの設定

1. `index.html` をブラウザで開く
2. 左パネル下部の「Google Sheets設定」セクションを見つける
3. 「Google Apps Script Web App URL」にコピーしたURLを貼り付け
4. 「設定を保存」ボタンをクリック
5. ページが自動的に再読み込みされます

### 5. 動作確認

1. タスクを1つ追加してみる
   - タスク名: 「テストタスク」
   - 期限: 明日の日付
   - 重要・緊急にチェック
   - 「保存」をクリック
2. Google Sheetsに戻って「タスク」シートを確認
3. タスクが自動的に追加されていることを確認

## スプレッドシートの構造

Google Apps Scriptが自動的に「タスク」という名前のシートを作成し、以下の列を設定します：

| 列 | 項目名 | データ型 | 説明 |
|----|--------|----------|------|
| A | ID | テキスト | タスクの一意識別子 |
| B | タスク名 | テキスト | タスクの名前 |
| C | 期限 | 日付 | タスクの期限（YYYY-MM-DD形式） |
| D | 登録日 | 日付 | タスクの登録日（YYYY-MM-DD形式） |
| E | 重要 | ブール値 | 重要フラグ（TRUE/FALSE） |
| F | 緊急 | ブール値 | 緊急フラグ（TRUE/FALSE） |
| G | 詳細 | テキスト | タスクの詳細情報 |

## 機能

### 自動同期

- **タスク追加**: アプリでタスクを追加すると、即座にGoogle Sheetsに保存されます
- **タスク編集**: タスクを編集すると、Google Sheetsの対応する行が更新されます
- **タスク削除**: タスクを削除すると、Google Sheetsから行が削除されます
- **初回読み込み**: アプリを開くと自動的にGoogle Sheetsからデータを読み込みます

### マルチデバイス対応

- スマートフォン、タブレット、PCなど、どのデバイスからアクセスしても同じデータが表示されます
- 複数人で同じスプレッドシートを共有すれば、チームでタスク管理ができます

## よくある質問

### Q: データはどこに保存されますか？

A: すべてのタスクデータはGoogle Sheetsに保存されます。ローカルストレージには設定情報（Web App URLとソート設定）のみが保存されます。

### Q: 複数人で使用できますか？

A: はい。スプレッドシートを他のユーザーと共有すれば、複数人でタスクを管理できます。ただし、全員が同じWeb App URLを使用する必要があります。

### Q: オフラインで使用できますか？

A: いいえ。このアプリはGoogle Sheetsに接続する必要があるため、インターネット接続が必要です。

### Q: セキュリティは大丈夫ですか？

A: Google Apps ScriptはGoogleのサーバー上で実行され、通信はHTTPSで暗号化されます。ただし、Web App URLを知っている人は誰でもデータにアクセスできるため、URLは慎重に管理してください。

### Q: デプロイしたWeb Appを更新するには？

A: 
1. Apps Scriptエディタでコードを変更
2. 「保存」をクリック
3. 「デプロイ」→「デプロイを管理」を選択
4. 鉛筆アイコンをクリック
5. バージョンを「新しいバージョン」に変更
6. 「デプロイ」をクリック

## トラブルシューティング

### 「データの取得に失敗しました」エラー

**原因と対処法**:
1. Web App URLが正しく設定されているか確認
2. Google Apps Scriptが正しくデプロイされているか確認
3. スプレッドシートが削除されていないか確認
4. ブラウザのコンソールでエラーメッセージを確認

### 「タスクの保存に失敗しました」エラー

**原因と対処法**:
1. Web App URLが正しいか確認
2. Google Apps Scriptの実行権限を確認
3. スプレッドシートのアクセス権限を確認

### タスクが表示されない

**原因と対処法**:
1. 「同期」ボタンをクリックして再読み込み
2. ブラウザをリフレッシュ
3. Web App URLが正しく設定されているか確認

### 初回デプロイで「このアプリは確認されていません」と表示される

**対処法**:
1. 「詳細」をクリック
2. 「（プロジェクト名）（安全ではないページ）に移動」をクリック
3. 「許可」をクリック

これは自分で作成したスクリプトなので安全です。

## 制限事項

### Google Apps Scriptの制限

- 1日あたりの実行時間: 90分（無料アカウント）
- 同時実行数: 30（無料アカウント）
- URLフェッチコール: 20,000回/日（無料アカウント）

通常の個人使用では、これらの制限に達することはありません。

## サポート

問題が解決しない場合は、以下を確認してください：

1. ブラウザのコンソールログ（F12キーで開く）
2. Google Apps Scriptの実行ログ
3. Google Sheetsのアクセス権限

それでも解決しない場合は、GitHubのIssueで報告してください。
